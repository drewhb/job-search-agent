# Job postings — seen registry

> Managed by the `scan-jobs` skill. Every URL surfaced in an inbox digest is recorded here so it is not shown again on subsequent scans.
>
> **To force a posting to re-surface:** delete its row.
> **To permanently suppress a URL:** add a row manually with **Rating when seen** set to `Suppressed`. Rows with this rating are always skipped, regardless of age.
> **Re-surface rule:** only postings whose `Date last surfaced` was >30 days ago and that were rated `Low` or `Medium` are eligible to be flagged as `(re-surfaced)` if still live. When re-surfaced, `Date last surfaced` is updated so the posting does not reappear on the next scan.
