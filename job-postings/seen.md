# Job postings — seen registry

> Managed by the `scan-jobs` skill. Every URL surfaced in an inbox digest is recorded here so it is not shown again on subsequent scans.
>
> **To force a posting to re-surface:** delete its row.
> **To permanently suppress a URL:** add a row manually and set **Rating when seen** to `High` so it is always skipped.
> **Re-surface rule:** only postings first seen >30 days ago that were rated `Low` or `Medium` are eligible to be flagged as `(re-surfaced)` if still live — rows rated `High` are not re-surfaced.
